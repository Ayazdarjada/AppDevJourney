# Contributing to AppDevJourney

## Welcome
Thanks for your interest in contributing! We welcome all sorts of contributions.

## How to Report Bugs or Request Features
- Check the existing issues to avoid duplicates.
- Open a new issue if needed with a detailed description.

## Pull Request Process
- Fork the repository.
- Create a new branch for your feature/fix (`git checkout -b feature-name`).
- Write clear, atomic commits.
- Submit a pull request (PR) to the `main` branch.
- PR will be reviewed and merged after approval.

## Coding Style and Guidelines
- Follow existing code style (indentation, naming).
- Write comments for complex code sections.
- Keep commit messages concise and descriptive.
